How To Host Website Using Flask-Nginx-Gunicorn

In this post, we will see how to host flask applications using gunicorn and nginx as a reverse proxy and static files server.
Lets follow this steps below:

sudo apt update

Now let's install python3, python3-pip, and Nginx, Use following command:

sudo apt install python3-pip python3-dev nginx

Let's install virtualenv

sudo pip3 install virtualenv

Lets create flask app directory.

mkdir myFlaskApp && cd myFlaskApp

Lets activate the virtual environment, before that lets name the virtual environment as "env"

(Remeber always activate virtual environment before starting any flask project.)

virtualenv env

Now, lets activate the virtual environment.

source env/bin/activate

As we've activated virtual environment; Lets install dependencies required for our flask app.

pip3 install flask gunicorn

If you already have flask app in your local device; then append all the required dependencies in "requirements.txt" file. Use following command in your local device.

pip3 freeze>requirements.txt

Now to install those dependencies on server; first upload that file on server by using filezilla (we have separate tutorial for that); and then run the following command on server: 

pip3 install -r requirements.txt

First let me create my python file which will have all the flask data.

sudo nano app.py

Paste the following standard content in app.py;

(Incase you have your own python flask file upload it on server.)

from flask import Flask
app = Flask(__name__)
@app.route('/')
def hello_world():
    return "Hello World!"
if __name__ == '__main__':
    app.run(debug=True,host='0.0.0.0')

Now, press 'Ctrl+S' and then to exit press 'Ctrl+X'

Next, we’ll create a file that will serve as the entry point for our application. This will tell our Gunicorn server how to interact with the application.

sudo nano wsgi.py

Copy the contents below to wsgi.py

from app import app

if __name__ == "__main__":
    app.run()

Now, press 'Ctrl+S' and then to exit press 'Ctrl+X'

Lets test gunicorn's ability to serve our project by running the command below:

gunicorn --bind 0.0.0.0:5000 wsgi:app

This will temporarily host your website on IP:5000

Terminate this with 'Ctrl+C'

Now deactivate the virtual environment:

deactivate

Lets now create a systemd service using the following commands:

sudo nano /etc/systemd/system/app.service

Paste the following content in this file:

(Carefully edit the information in this.)

[Unit]
#  specifies metadata and dependencies
Description=Gunicorn instance to serve myproject
After=network.target
# tells the init system to only start this after the networking target has been reached
# We will give our regular user account ownership of the process since it owns all of the relevant files
[Service]
# Service specify the user and group under which our process will run.
User=prasad
# give group ownership to the www-data group so that Nginx can communicate easily with the Gunicorn processes.
Group=www-data
# We'll then map out the working directory and set the PATH environmental variable so that the init system knows where our the executables for the process are located (within our virtual environment).
WorkingDirectory=/home/user/FlaskAppFolder/
Environment="PATH=/home/user/FlaskAppFolder/env/bin"
# We'll then specify the commanded to start the service
ExecStart=/home/user/FlaskAppFolder/env/bin/gunicorn --workers 3 --bind unix:app.sock -m 007 wsgi:app
# This will tell systemd what to link this service to if we enable it to start at boot. We want this service to start when the regular multi-user system is up and running:
[Install]
WantedBy=multi-user.target

Now, press 'Ctrl+S' and then to exit press 'Ctrl+X'

Lets activate this service:

sudo systemctl start app
sudo systemctl enable app

Now type:

ls

A file named app.sock will be automatically created. Folder structure so far:

myFlaskApp
  |____ static (if you've created a static folder)
  |____ templates (if you've created a temlates folder)
  |____ app.py
  |____ wsgi.py
  |____ env
  |____ app.sock

If "app.sock" is not autimatically created; Then
1. Check app.service file. Check each line in it.
2. Run the following command

sudo systemctl status app

Create a file named "app" inside /etc/nginx/sites-available:

sudo nano /etc/nginx/sites-available/app

Now paste the following content in this file:

(DO NOT FORGET TO CHANGE IP ADDRESS AND YOU FILE PATH)

server {
listen 80;
server_name 165.232.177.116;

location / {
  include proxy_params;
  proxy_pass http://unix:/home/user/FlaskAppFlder/app.sock;
    }
}

Activate this configuration by executing this:

sudo ln -s /etc/nginx/sites-available/app /etc/nginx/sites-enabled

If you do changes in "/etc/nginx/sites-available/app" file or if you are re-running the above command and gets error; Then run:

sudo ln -sf /etc/nginx/sites-available/app /etc/nginx/sites-enabled

Restart nginx and your website should work fine!

sudo systemctl restart nginx

Visit http://165.232.177.116/

If you are using a firewall (for eg. ufw), don't forget to allow nginx to receive incoming traffic on port 80

sudo ufw allow 'Nginx Full'

If you get Error 502 like below on web brower

Then Follow few stes:
1) Check if app.sock file is present or not.
2) Check app.py file have error or not by running

sudo systemctl status app

3) Check if you have installed dependencies inside and outside of virtual environment.
4) If above status command doesn't show the actual error then run python file separately:

gunicorn --bind 0.0.0.0:5000 wsgi:app

If you do changes in Python file; everytime run:

sudo service app restart

And If you do changes in Python file as well as other configuration files; everytime run:

sudo service nginx restart
sudo service app restart
